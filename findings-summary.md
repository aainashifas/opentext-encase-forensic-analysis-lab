# Findings Summary

## File Signature Analysis
File signature analysis showed how forensic tools can detect mismatches between file extensions and actual file headers. This is important because renamed files may hide their true format.

## Hash Analysis
MD5 and SHA-1 hashes were generated for applicable files. Hashing supports evidence integrity verification, known-file comparison, and filtering of irrelevant files.

## Registry Artifacts
Registry artifacts, including NTUSER.DAT, provided insight into user-specific activity such as recently accessed files, typed URLs, application usage, and system interaction.

## Email Artifacts
Email artifact review demonstrated how forensic tools can recover and organize email archives, deleted messages, and related conversations.

## Internet Artifacts
Internet artifacts showed browser-related evidence such as visited URLs, cached content, cookies, and timestamps.

## Keyword Searching
Indexed and raw keyword searches were used to locate relevant terms across evidence sources. Indexed search provided faster querying, while raw keyword search allowed deeper pattern-based searching.

## Bookmarking and Tagging
Bookmarks and tags were used to organize evidence items for later review and reporting. This supports repeatability and improves the structure of forensic documentation.
